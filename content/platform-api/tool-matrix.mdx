This matrix helps distinguish between:

- examples confirmed directly by backend tests or explicit code paths
- examples inferred from published frontend/backoffice behavior

Use the tool-specific Swagger UI page as the runtime source of truth when the published schema changes.

## Recommendation

When documenting or demoing a tool:

1. Start from this matrix.
2. Open the tool-specific Swagger UI page.
3. Copy the currently published field names from the runtime schema.
4. Keep Mintlify examples high-value and stable, not exhaustive.
