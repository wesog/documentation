The `sort_by` and `sort_order` parameters of the search endpoint (`GET /imgs`) define the sorting criteria to rank the image results.

## Sort By

The `sort_by` parameter specifies the criterion used to sort the search results. There are 4 different types of sorting criteria:

1. **Similarity**: `sort_by=similarity` (or just `sort_by=sim`). Note that this sorting option requires one of the following filters on the query:
   - `t=`: Full text search.
   - `t~`: Semantic text search.
   - `i=`: Reverse identity image search.
   - `i~`: Reverse image semantic search.
   - `p~`: Person face. (Similarity sorting between query face and faces on the image).
2. **Numerically**:
   - `sort_by=id`: Image identifier (This is the default).
   - `sort_by=width`: Image width.
   - `sort_by=height`: Image height.
   - `sort_by=n_faces`: Number of faces in the image.
3. **Date**
   - `sort_by=date`
4. **Alphabetically**:
   - `sort_by=src`
   - `sort_by=headline`
   - `sort_by=caption`
   - `sort_by=location`
   - `sort_by=country`
   - `sort_by=city`
   - `sort_by=author`
   - `sort_by=medium`
   - `sort_by=technique`

## Sort Order

The `sort_order` parameter defines the order (ascending or descending) of the sorted results.

- `sort_order=asc` (This is the default)
  - For similarity sorting, this orders from **most** to **least similar**.
  - For numerical sorting, this orders from **smallest** to **largest**.
  - For date sorting: from the **earliest** to the **most recent**.
  - For alphabetical sorting, this orders from **A** to **Z**.
- `sort_order=desc`
  - For similarity sorting, this orders from **least** to **most similar**.
  - For numerical sorting, this orders from **largest** to **smallest**.
  - For date sorting: from the **most recent** to the **earliest**.
  - For alphabetical sorting, this orders from **Z** to **A**.
