Wesog documentation is split into two complementary layers:

1. **Human documentation in Mintlify**
2. **Interactive API references generated from Core OpenAPI**

The Mintlify site is the editorial entrypoint for customers, partners and new tenants. It explains what each product area does, how to integrate it, and where to find examples and operational guidance.

The interactive API references are served by the Core API and are the canonical technical contract for developers.

## Source of truth

The source of truth for the API contract is **not** the static `openapi.yaml` snapshot stored in this repository.

The source of truth is the OpenAPI generated by Core:

- Public platform OpenAPI: `https://api.wesog.com/api/public/openapi.json`
- Public interactive reference: `https://api.wesog.com/api/public/docs`

## Domain references

Wesog follows a **private-first** policy.

Only the domains that provide clear integration value to customers should be exposed publicly. Right now the public domain references should be treated as:

- Tools Swagger UI: `https://api.wesog.com/api/domains/tools/docs`
- Tools OpenAPI: `https://api.wesog.com/api/domains/tools/openapi.json`
- Workflows Swagger UI: `https://api.wesog.com/api/domains/workflows/docs`
- Workflows OpenAPI: `https://api.wesog.com/api/domains/workflows/openapi.json`

Everything else should default to authenticated or internal-only documentation until there is a strong product reason to publish it.

## Tool references

Each published tool can also expose its own public technical reference:

- Tool Swagger UI: `https://api.wesog.com/api/tools/caption/docs`
- Tool OpenAPI: `https://api.wesog.com/api/tools/caption/openapi.json`

Those routes are public for reading, but trying requests from the interactive console still requires a valid tenant API key.

## Local development

When running locally, the equivalent endpoints are usually:

```text
http://127.0.0.1:3333/api/public/openapi.json
http://127.0.0.1:3333/api/public/docs
http://127.0.0.1:3333/api/domains/tools/docs
http://127.0.0.1:3333/api/tools/caption/docs
```
