The `sort_by` and `sort_order` parameters of the search endpoint (`GET /imgs`) define the sorting criteria to rank the image results.

## Sort By

The `sort_by` parameter specifies the criterion used to sort the search results. There are three different types of sorting criteria:

- **Similarity**:
  - `similarity` (alias `sim`). Note that this sorting option requires one of the following filters on the query:
    - `t=`: Full text search
    - `t~`: Semantic text search
    - `i=`: Revesre identity image search
    - `i~`: Revesre image semantic search
    - `p~`: Person face
- **Numerically**:
  - `id` (This is the default)
  - `width`
  - `height`
  - `n_faces`
- **Date**
  - `date`
- **Alphabetically**:
  - `src`
  - `headline`
  - `caption`
  - `location`
  - `country`
  - `city`
  - `author`
  - `medium`
  - `technique`

## Sort Order

The `sort_order` parameter defines the **ascending** or **descending** order of the sorted search results.

- `sort_order=asc` (This is the default)
  - For similarity sorting, this orders from **most** to **least similar**.
  - For numerical sorting, this orders from **smallest** to **largest**.
  - For date sorting: from the **earliest** to the **most recent**.
  - For alphabetical sorting, this orders from **A** to **Z**.
- `sort_order=desc`
  - For similarity sorting, this orders from **least** to **most similar**.
  - For numerical sorting, this orders from **largest** to **smallest**.
  - For date sorting: from the **most recent** to the **earliest**.
  - For alphabetical sorting, this orders from **Z** to **A**.
