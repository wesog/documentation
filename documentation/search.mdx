## Introduction

The search endpoint `GET /imgs` is the most powerful API endpoint, allowing the creation of complex queries using a custom syntax. This page explains how to use the Wesog Search API to construct advanced search queries.


### Input: Request Parameters

The search endpoint (`HTTP GET /imgs`) has the following URL parameters:

- `q`: Query Filtering Formula. See more info [here](https://wesog.mintlify.app/documentation/search-filters). **Required**.
- `sort_by`: Sorting criteria (Default: `id`)
- `sort_order`: `asc` for ascending, `desc` for descending (Default: `asc`)
- `page`: Page number. First page is 1 (Default: `1`)
- `limit`: The number of images per page (default: `50`)
- `fields`: The returning image fields (comma-separated) (default: `id`)
- `facets`: 'Faceted Search. Facets (comma separated) that will appear in the search result. 20 is the defualt max number of options per facet.
- `count`: If true, includes the exact number of images in the search results. (default: `false`)
- `lang`: The language used the text search (default: `en`)


### Output: Returned Response

The response is a JSON object containing the found images with the specified `fields` and additional information such as the total number of images found and the time taken.


```json
// HTTP GET /imgs?q=t~us elections&sort_by=t~us elections&fields=id,headline
{
  "images": [
    {"id": 323, "headline": "Joe Biden drops his candidacy for president of the United States"},
    {"id": 324, "headline": "Donald Trump getting shot in the ear"}
  ],
  "num_images": 2,
  "facets": {},
  "time_taken": "0.02s"
}
```
